# 無大台香港典藏館 DLLM-Library

「無大台香港典藏館」以開源方式賦能海外香港社群，讓每個地區都能獨立運行屬於自己的文化保存平台。通過公開透明的代碼與去中心化的架構，我們希望每一本書、每件藏品都能找到它的歸宿，共同守護香港的文化記憶。

## 項目背景及目的

「無大台香港典藏館」是一個開源的社群平台，專為海外香港人設計，旨在促進珍貴圖書及文化藏品的保存與傳承。鑑於香港人移居海外後，個人藏品可能因空間限制或下一代興趣轉移而面臨丟棄風險，本項目希望通過去中心化的方式，讓全球香港社群：

- **上架與分享** 個人珍藏的圖書或文化物品，與同好交流；
- **借閱或轉讓** 藏品給真正珍惜的人，避免文化遺產流失；
- **社群自治** 不同地區的香港社群可自行搭建伺服器，靈活管理本地化平台。

作為開源項目，源代碼公開於 GitHub，允許任何海外香港社群自由下載、修改及部署，確保「無大台」的精神貫徹，無需依賴單一中央伺服器。

## 已實現功能

以下為平台目前已完成並上線的功能：

### 用戶系統 (User System)

- 透過 **Firebase Authentication** 註冊及登入，支援 Email 認證。
- 用戶個人檔案包含暱稱、地址（僅顯示大致地區）、多種聯繫方式（WhatsApp、Signal、Telegram、Email）。
- **角色管理**：支援普通用戶（User）、管理員（Admin）、版主（Moderator）及交收點管理員（Exchange Point Admin）。
- 用戶可置頂最多 5 件藏品於個人頁面。

### 藏品管理 (Item Management)

- 藏品完整的新增、編輯、刪除功能。
- 支援多種狀態：可借閱（Available）、可交換（Exchangeable）、可贈送（Gift）、已預留（Reserved）、已轉讓（Transferred）。
- 藏品狀態標記（全新、近全新、良好、一般、較差）。
- **圖片上傳**：透過 Google Cloud Storage 簽名 URL 上傳，伺服器端自動壓縮及生成縮略圖。
- **NSFW 內容檢測**：客戶端上傳前自動檢測不當影像內容。
- **多維度搜尋**：按關鍵字、分類、狀態、地區搜索，支援地理空間查詢（Geospatial Search）。
- **分類系統**：多層級分類樹，支援中英雙語分類標籤，提供熱門分類、最近更新分類及預設分類。
- **藏品推薦**：四種推薦類型——用戶精選、最新上架、熱門藏品、管理員推薦。

### 交易系統 (Transaction System)

- 完整的借閱 / 轉讓流程：**申請 → 批准 → 交接 → 確認收取**，支援取消及過期處理。
- **快速交易**：支援面對面即時交收，簡化審批流程。
- 多種交收地點選項：持有者地址、申請者地址、公共交收點、面對面。
- 交易狀態追蹤及流程圖展示。
- 支援以 QR Code 分享交易資訊。
- 交收確認上載收據圖片。

### 交收點 (Exchange Points)

- 公共交收點目錄及列表。
- 交收點管理員角色，獨立管理交收點資訊。
- 地圖顯示交收點位置。

### 新聞 / 公告系統 (News System)

- 管理員及版主可發佈、編輯及隱藏新聞公告。
- 支援 Markdown 格式撰寫內容。
- 新聞可關聯相關藏品及標籤。
- 最新新聞橫幅展示。

### 評論系統 (Comments)

- 用戶可對藏品發表、編輯及刪除評論。
- 支援分頁瀏覽，可按藏品或用戶查詢評論紀錄。

### 地圖及地理定位 (Map & Geocoding)

- 透過 **Google Maps API** 進行地址正向地理編碼。
- 以 **Leaflet** 在客戶端渲染互動地圖。
- 基於 **Geofire** 的地理空間藏品搜索。

### 批量匯入 (Bulk Import)

- 支援 JSON 格式批量匯入藏品。
- **GoodReads 匯入**：專屬頁面從 GoodReads 匯入書籍資料。
- 匯入前自動檢測重複標題。

### 國際化 (i18n)

- 完整支援**繁體中文**及**英文**雙語介面。
- 透過 i18next 實現語言切換，自動偵測瀏覽器語言。
- 藏品分類支援雙語標籤。

### SEO 與伺服器端渲染 (SSR)

- 搜索引擎爬蟲自動偵測，為重要頁面（首頁、藏品詳情、用戶檔案、交易詳情）提供伺服器端渲染。
- 優化社交媒體分享預覽（Open Graph）。

### 其他功能

- 新用戶引導導覽（Onboarding Tour）。
- 啟動畫面（Splash Screen），可由管理員配置。
- 標籤雲視覺化展示分類分佈。
- 平台管理設定（聊天連結、關於我們、啟動畫面等）。

## 技術架構

| 層級         | 技術                                                                            |
| ------------ | ------------------------------------------------------------------------------- |
| **前端**     | React 19、TypeScript、Vite、Material UI 7（MUI）、Apollo Client、React Router 7 |
| **後端**     | Node.js 22、Express、Apollo Server 4、TypeScript                                |
| **API**      | GraphQL（28 個查詢 + 22 個變更操作）                                            |
| **數據庫**   | Firebase Firestore                                                              |
| **認證**     | Firebase Authentication                                                         |
| **儲存**     | Google Cloud Storage（簽名 URL 上傳）                                           |
| **託管**     | Firebase Hosting + Firebase Cloud Functions                                     |
| **地圖**     | Leaflet（客戶端）、Google Maps Services（伺服器端地理編碼）                     |
| **圖片處理** | Sharp（伺服器端壓縮）、NSFW.js（客戶端內容審核）                                |
| **國際化**   | i18next（繁體中文 + 英文）                                                      |
| **代碼生成** | GraphQL Codegen（前後端共用 Schema）                                            |
| **測試**     | Vitest（前端）、Jest（後端）                                                    |

### 架構特點

- **單一 Cloud Function**：一個 Firebase Cloud Function 同時提供 GraphQL API 及 SSR 頁面服務。
- **共用 Schema**：前後端透過 `graphql-codegen` 從共同的 `schema.graphql` 自動生成類型定義。
- **SPA + SSR 混合**：一般用戶訪問 SPA，搜索引擎爬蟲自動獲取 SSR 頁面。
- **GitHub Repository**：包含完整源代碼、安裝指引及 API 文檔，方便開發者貢獻。
- **安全性**：採用 HTTPS 加密、Firebase Auth Token 認證、伺服器端權限驗證。

## 開源運作模式

- **去中心化原則**：本項目不依賴單一伺服器，鼓勵各地區香港社群自行搭建獨立實例（Instance）。例如，倫敦社群可運行自己的 Firebase 項目，獨立管理本地用戶及藏品數據。
- **源代碼公開**：所有代碼（前端、後端及 GraphQL Schema）公開於 GitHub，採用 MIT License，允許自由使用、修改及分發。
- **部署靈活性**：提供詳細的[部署指南](./docs/SETUP.cn.md)，基於 Firebase 平台，方便社群快速搭建。
- **私隱保護**：用戶資料（特別是地址）不公開，伺服器管理員需遵守基本數據保護指引（參考 GDPR）。
- **免費使用**：開源模式確保無需付費，社群可透過 Firebase 免費額度運行，或自行承擔運營成本。

## 社群參與

- **貢獻方式**：歡迎全球開發者及香港社群參與，提交 Pull Requests 改進功能、翻譯介面或修復 Bug。
- **本地化支持**：提供 i18next 多語言框架，方便社群將 App 翻譯成其他語言（如日語、德語）以融入當地香港人網絡。
- **討論渠道**：設立 Discord 或 Matrix 群組，供開發者及用戶交流技術問題及建議。

## 未來展望

平台持續收集社群反饋，未來可能的方向包括：

- 跨伺服器數據共享協議，實現全球藏品搜索；
- 電郵通知系統，自動推送交易狀態更新；
- 全文搜索引擎，提升藏品搜尋體驗；
- WebSocket 即時訂閱，實現借閱狀態即時更新；
- 提供插件系統，讓社群自定義功能（如活動通知、藏品展覽）。
